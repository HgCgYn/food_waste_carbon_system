【摘要】
隨著全球對於永續發展與溫室氣體減排的關注日益增加，減少食物浪費（Food Waste）已成為企業 ESG 與碳足跡盤查的重要指標。本研究旨在解決現有廚餘辨識系統在實際應用情境下準確率不足的痛點。傳統以 YOLO 為基礎的邊緣運算物件偵測模型，受限於參數量與訓練資料的侷限性，在面對形狀相似、遮擋嚴重或非典型飲食文化（如西式餐點）的廚餘場景時，極易產生誤判或漏判。為突破此一技術瓶頸，本研究提出了一種創新的「協同辨識架構」，結合邊緣端高效的 YOLO 模型與雲端大型視覺模型（VLM, 如 GPT-4o 與 Gemini 2.5 Flash）。系統首先利用 YOLO 進行快速初篩，當判斷信心度低於閾值（0.70）時，自動觸發雲端 VLM 進行二次視覺糾錯。研究導入全圖脈絡提示（Red Box Context）、多標籤輸出與思維鏈（Chain-of-Thought, CoT）技術。實驗結果顯示，相較於純 YOLO 模型的 26.5% 準確率，導入 GPT-4o 後準確率顯著提升至 44.1%（提升幅度達 17.6%），且進階統計檢定（McNemar's Test, p = 0.0247）亦證實此提升具備統計顯著性。本系統展現了高度的商用價值，特別適用於「吃到飽餐廳」的自動化剩食罰款應用，有效減少人工結帳爭議並推動餐飲業的淨零轉型。

【關鍵詞】廚餘辨識、物件偵測、大型視覺模型、思維鏈、吃到飽餐廳

1. 前言

1.1 研究背景與動機
全球剩食問題（Food Waste）不僅造成龐大的經濟損失，更是溫室氣體排放的重要來源之一。根據聯合國糧農組織（FAO）的統計，全球每年有三分之一的食物被浪費，其所產生的碳足跡對環境造成了不可忽視的負擔。近年來，隨著企業 ESG（環境、社會、公司治理）意識抬頭，越來越多的餐飲業者、企業團膳以及醫療機構開始重視剩食的量化與減量管理。特別是在「吃到飽（All-You-Can-Eat）」餐廳的商業模式中，顧客因過量夾取而導致的高單價食材浪費情況屢見不鮮。許多業者雖訂有「剩食罰款」的規章，但礙於人工結帳時的判斷主觀性與顧客爭議，往往難以嚴格執行。因此，開發一套客觀、自動化且高準確率的廚餘辨識系統，不僅具有深遠的環保意義，更具備龐大的商業潛力。

目前市面上的廚餘辨識系統多半採用以邊緣運算（Edge Computing）為主體的輕量化深度學習模型，如 YOLO (You Only Look Once) 系列。這類模型具備推論速度快、建置成本低的優勢，非常適合部署於餐廳餐盤回收區的樹莓派或工業電腦中。然而，廚餘的視覺特徵極度複雜，食物在食用後往往會變形、破碎或混合，加上亞洲便當菜色繁多且常常互相遮擋，傳統 YOLO 模型在缺乏足夠訓練樣本的情況下，其辨識準確率往往不盡理想。當面對外型相似的食材（如將蒸蛋誤判為豆腐）或未曾見過的外國菜色時，單靠 YOLO 的特徵提取能力已遇到明顯的效能瓶頸。

1.2 研究目的
為了解決傳統邊緣運算模型在複雜廚餘場景下的辨識痛點，本研究提出了一套「邊緣與雲端協同（Edge-Cloud Collaborative）」的混合式架構。本研究的主要目的包括：
1. 建立一套自動觸發機制，利用 YOLO 進行快速初篩，並在信心度不足時引入大型視覺模型（VLM）進行視覺糾錯。
2. 探討並優化 VLM 的提示工程（Prompt Engineering），包括全圖脈絡提示（Red Box Context）與思維鏈（Chain-of-Thought, CoT）推理，以極大化 VLM 的糾錯能力。
3. 透過嚴謹的實驗設計與進階統計檢定（如 McNemar's Test、卡方檢定），量化評估純 YOLO 模式與 YOLO+VLM 模式在不同難易度及不同飲食文化（東方 vs 西方）場景下的準確度差異。
4. 探討本系統在吃到飽餐廳等真實商業場景中的落地應用價值。

**系統辨識限制說明：**
讀者在實際體驗時需注意，本系統目前在設計上**僅支援放置於「圓形餐盤」上的食物辨識**。此限制主要是基於以下工程考量：
1. **YOLO 訓練特徵侷限**：本研究在初期訓練輕量化 YOLO 模型時，所蒐集的資料集高度集中於標準圓形餐盤上的食物狀態。若將食物放置於方形餐盒或不規則容器中，會導致背景特徵與訓練分佈脫節，大幅降低初篩信心度。
2. **空間定位與裁切邏輯**：系統在進行 Red Box 全圖脈絡提示時，依賴圓形餐盤的邊緣輪廓作為空間定位的基準參考（Reference Anchor），這使得 VLM 在理解「食物殘渣與盤子間的相對比例」時更具方向感。若缺乏標準圓盤作為參照物，VLM 在評估食物份量與種類時的幻覺率會顯著提高。

2. 相關文獻與技術探討

2.1 YOLO 物件偵測技術於邊緣運算之應用
YOLO（You Only Look Once）模型自提出以來，因其端到端（End-to-End）的偵測架構與卓越的實時推論速度，成為工業瑕疵檢測、智慧交通與餐飲辨識等領域的主流選擇。在本專案中，我們採用了最新的 YOLOv11 架構。YOLO 透過卷積神經網路（CNN）從影像中萃取特徵，並直接預測物件的邊界框（Bounding Box）與類別機率。然而，對於「廚餘」這種非剛性、高變異性的特徵，輕量化 YOLO 模型的參數量難以涵蓋世界各地的各種飲食文化。當模型遇到未充分訓練的類別或受到光線、遮擋影響時，信心度（Confidence Score）會急遽下降，這正是邊緣運算模型常見的泛化能力（Generalization）限制。

2.2 大型視覺模型（Large Vision Models, VLM）的崛起
近年來，大型語言模型（LLM）結合視覺編碼器（Vision Encoder）演進而成了大型視覺模型（VLM），如 OpenAI 的 GPT-4o 以及 Google 的 Gemini 2.5 Flash。這些模型依賴數以百億計的參數與海量的跨網域圖文配對資料進行訓練，因而具備了強大的「廣泛世界知識（World Knowledge）」與「零樣本推理（Zero-Shot Reasoning）」能力。與傳統需要大量標註資料重新訓練的 YOLO 不同，VLM 能夠直接理解自然語言提示（Prompt），並對未見過的菜色或複雜的食物殘渣進行上下文推理。然而，VLM 的缺點在於 API 呼叫成本較高且具有網路延遲，不適合對影片流進行每秒 30 幀的連續偵測。因此，將 YOLO 作為前置過濾器，VLM 作為後端的智慧大腦，成為了兼顧效能與成本的最佳解。

2.3 提示工程與思維鏈（Chain-of-Thought, CoT）
在引導 VLM 進行視覺任務時，提示工程（Prompt Engineering）的設計至關重要。早期的視覺提示通常直接詢問模型「圖片中是什麼物體」，這容易導致模型產生幻覺（Hallucination）或因視覺捷徑（Visual Shortcuts）而做出草率的判斷。思維鏈（Chain-of-Thought, CoT）是一種啟發模型邏輯推理的技巧，透過在提示中強制要求模型「分步思考」。在本研究中，我們要求 VLM 在給出最終食物標籤前，必須先輸出一段「分析（Analysis）」，描述物件的顏色、紋理與形狀特徵。這種先觀察後判斷的機制，被證實能大幅降低 VLM 將視覺相似物體互相混淆的機率。
